Une fonction : 
1- encapsuler du code.
2- éviter la répétition.
3- l'abstraction.
4- Permet la modularité et la réutilisation du code.
5- facilite la maintenance du code.

paradigme de programmation : Procédural 

Instruction vs Expression

Instruction : Suite d'opérations qui ne retourne jamais quelque chose.
Expression : Suite d'opérations qui vont toujours retourner quelque chose.

oppérateur "=" est une expression MAIS PAS en py (c'est une instruction).

Opérateur walrus ":=" c'est une assignation sous forme d'expression.
	Lambda exemple de code avec l'opérateur walrus:
	speed = 0
	a = lambda v : (speed := v)
	a(3)


La docstring : immédiatement après la déclaration de la fonction.

Les types de fonctions qui ne retournent rien : 
	Une fonction While True:
	Une levée d'exception ne retourne rien.
	Quelques fonctions utilitaires ne retournent rien (forcer la sortie immédiate du programme)


Les paramètres passés aux fonctions sont des variables locales, les variables locales n'existent plus à la sortie de la fonction. 


La surcharge de fonctions en python :
def f(a,b,c):
f(3,4,5) -> lors de l'appel, appel avec argument positionnel
f(a=3,b=4, c=5) -> appel avec argument par mots clés, pas obligé de respecter l'ordre. 
Possible aussi de les mélanger -> f(4,c=3,b=1)


exemple avec les listes :
	def f(v, li=[]):
		li.append(v)
		print(li)
		return li
	a =[1,2]
	f(5,a) #[1,2,5]
	a = f(6,a) #[1,2,5,6]
	b = f(7) #[7]
	a = f(10,b) #[7,10]
	c = f(-1) #[7,10,-1]
	
La liste vide existe dès la définition de fonction (c'est la même liste). 
La valeur par défaut doit être immuable (quelque chose qui ne peut pas changer -> tuple, entier, float, string)


forcer l'utilisation du positionnel / mot clé:
def f(a , b, /, c, *, d, e)
	Tous ce qui est à gauche du "/," doit être positionnel.
	Tous ce qui est à droite du "*," doit être par mot clé.
