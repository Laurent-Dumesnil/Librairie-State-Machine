Module tracking_device

Module de gestion de dispositifs de suivi basé sur l'approche de programmation en flux tiré (pull-based programming).

Ce module permet le développement d'une application exploitant une architecture de suivi de composants utilisant un double paradigme de programmation :

  • Programmation orientée objet (OOP) : Les composants respectent une interface de programmation polymorphisme permettant une généralisation et une simplification du développement logiciel.
  • Programmation en flux tiré : À la différence de la programmation évènementielle, le suivi de l'état est initié de manière explicite par des appels réguliers aux méthodes de suivi réalisé dans une boucle de contrôle principale.

L'architecture du module repose sur ces trois classes fondamentales :

  • TrackingDevice : Une classe abstraite représentant un dispositif de suivi générique. Elle gère l'état, les sous-dispositifs et la logique de suivi.
  • TrackingManager : Une classe permettant la gestion de plusieurs dispositifs de suivi.
  • TrackingApplication : Une classe spécialisée de TrackingManager qui permet de gérer l'application de suivi dans son ensemble en réalisant la boucle principale et la synchronisation des dispositifs.

Utilitaire :

  • TriggerDevice : Une classe concrète dérivée de TrackingDevice permettant le déclanchement une action périodique à un intervalle spécifié, facilitant ainsi la gestion d'actions automatiques basées sur le temps. En fait, cette classe est très intéressante car … à trouver par l'étudiant.

Exemple minimaliste :

from trigger_device import TriggerDevice

# Création d'un dispositif de suivi simple avec une action périodique
def action_1():
    print("Action 1 déclenchée!")

device_1 = TriggerDevice(duration=5.0, action=action_1, name="MyDevice1")
device_2 = TriggerDevice(duration=12.5, action=lambda : print("Action 2 déclenchée!"), name="MyDevice2")

# Simulation du suivi
app = TrackingApplication()
app.add_device(device_1)
app.add_device(device_2)

# Exécution de l'application
app.run_forever()

class TrackingApplication (device: OptionalOneOrMany[TrackingDevice] = None)

Une classe représentant une application exploitant les dispositifs TrackingManager.

Cette classe a pour objectif de fournir un cadre permettant de créer des applications qui effectuent des opérations de suivi en continu. Elle peut être utilisée pour gérer des TrackingDevice qui doivent fonctionner de manière ininterrompue tout en les synchronisant.

Cela permet de développer des solutions de suivi robustes capables de répondre à des besoins divers.

Elle hérite de TrackingManager et utilise un ElapsedTimer pour établire le temps écoulé.

var RunningCondition : TypeAlias

Un alias représentant une condition déterminant si la boucle de l'application doit cesser ou se poursuivre.

Cet alias est utilisé pour définir une objet exécutable (callable) déterminant la condition d'arrêt de la boucle principale réalisée par run_until. La fonction détermine si l'application termine ou non selon l'une de ces deux valeurs possibles :

  • None : L'application continue de s'exécuter (None, on ne s'arrête pas).
  • Any : Dès qu'une veleur différente de None est retournée, la boucle termine et l'application s'arrête. Dans ce cas, la valeur retournée est renvoyée par la méthode run_until.

Cet alias est défini par Callable[[], Any | None], un exécutable qui ne prend aucun argument et renvoie une valeur arbitraire ou None.

Cette approche permet de définir des conditions d'arrêt dynamiques et flexibles pour les applications de suivi.

Voici deux fonctions simples qui illustrent l'utilisation de cet alias :

```python
# Exemple d'utilisation de l'alias RunningCondition

from time import perf_counter
from tracking_application import TrackingApplication

# Exemple 1 : Arrêt après un nombre d'itérations
current_iteration: int = 0
max_iteration: int = 10000
def max_iteration_reached() -> Any | None:
    nonlocal current_iteration
    current_iteration += 1
    return f'{max_iteration} reached!' if current_iteration >= max_iteration else None

app = TrackingApplication()
# ...
app.run_until(max_iteration_reached)

# Exemple 2 : Arrêt après une durée prédéfinie
reference_counter: float = perf_counter()
duration: float = 120.0 # seconds
def total_duration_reached() -> Any | None:
    nonlocal reference_counter, duration
    return f'{duration} reached' if perf_counter() - reference_counter >= duration else None

app = TrackingApplication()
# ...
app.run_until(total_duration_reached)
```

def run_forever(self) ‑> NoReturn

Exécute l'application indéfiniment.

def run_until(self, running_condition: RunningCondition)

Exécute l'application jusqu'à ce que la condition fournie soit remplie.

La boucle continue jusqu'à ce que la condition renvoie une valeur autre que None. Chaque itération suit le temps et appelle la méthode track de tous les dispositifs.

Une fois la condition remplie (retour d'une valeur différente de None), elle arrête l'exécution et renvoie le résultat reçu par la condition.

Args :

  • running_condition (Callable[[], Any | None]) : La condition à remplir pour arrêter l'itération.

Returns :

  • Any : Le résultat renvoyé par la condition lorsqu'elle est remplie.

class TrackingDevice (name: str | None = None, enabled: bool = True)

Classe de base représentant un dispositif de suivi.

Pour maintenir son état, le dispositif doit être mis-à-jour par un appel formel de la méthode track. Ainsi, tous les dispositifs restent synchronisés les uns avec les autres s'ils sont tous maintenu à jour simultanément.

Un dispositif peut posséder des sous-dispositifs. Le patron de conception composite est utilisé pour que chaque dispositif puisse posséder et gérer des sous-dispositifs.

Un dispositif offre plusieurs outils de gestion : - un nom unique - la notion de validité/conformité - la notion de d'activation/désactivation - la possibilité de se réinitialiser - le suivi de son état - la gestion de ses sous-dispositifs.

Attributs : name (str | None) : Le nom du dispositif de suivi. enabled (bool) : Indique si le dispositif est activé. sub_devices_count (int) : Le nombre de sous-dispositifs gérés par ce dispositif de suivi.

Initialise une nouvelle instance de la classe TrackingDevice.

Args : name (str | None) : Le nom du dispositif de suivi. Par défaut, None. enabled (bool, optionnel) : Indique si le dispositif est activé. Par défaut, True.

prop valid : bool | str

Établie la validation/conformité du dispositif conditionnel à tous ses sous-dispositifs.

Le rôle de cette propriété a été modifié à cause de l'approche composite. Maintenant, pour être valide/conforme, un dispositif doit lui-même être valide ainsi que tous ses sous-dispositifs. Afin de rester le plus générique possible et d'éviter les répétitions de parcours, on rend cette substitution finale et on délègue la logique de validation locale à une autre fonction polymorphique : _do_valid.

Returns : bool | str : Le statut de validation. Si valide, retourne True. Sinon, retourne une chaîne décrivant le problème.

class TrackingManager (device: OptionalOneOrMany[TrackingDevice] = None)

La classe TrackingManager gère une collection d'instances de TrackingDevice, fournissant une interface pour ajouter, supprimer et gérer ces dispositifs collectivement. Elle permet de suivre, réinitialiser, et valider plusieurs dispositifs de manière synchronisée.
